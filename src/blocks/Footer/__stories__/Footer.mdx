## Overview

`Footer` renders a configurable page footer from the `footer-block` JSON block. The block is assembled from optional floors, so the same block can be used for anything from a compact legal footer to a full navigation footer with social links, language switching, and attribution.

### Floor map

| Floor | Purpose                                                         | Controlled by | Render condition                            |
| ----- | --------------------------------------------------------------- | ------------- | ------------------------------------------- |
| 1     | Navigation: logo + link columns                                 | `navigation`  | `navigation.columns` is provided            |
| 2     | Contacts: title + social icon links                             | `contacts`    | `contacts.links` contains at least one item |
| 3     | Disclaimer: YFM legal text                                      | `disclaimer`  | `disclaimer` is provided                    |
| 4     | Copyright: legal links, language switcher, logo, copyright text | `copyright`   | `copyright` is provided                     |
| 5     | Attribution: ŌĆ£Created with ...ŌĆØ strip                           | `attribution` | `attribution` is `true`                     |

The floors are rendered in the order above. `backgroundColor` and `indentTop` are applied to the root footer and affect all enabled floors.

## Parameters

`type: 'footer-block'`

## Root props

```ts
type FooterBlockProps = {
    navigation?: FooterNavigation;
    contacts?: FooterContacts;
    disclaimer?: FooterDisclaimer;
    copyright?: FooterCopyright;
    attribution?: boolean;
    backgroundColor?: ThemeSupporting<string>;
    indentTop?: '0' | 'xs' | 's' | 'm' | 'l' | 'xl';
};
```

| Prop              | Type                                       | Description                                                                                            |
| ----------------- | ------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
| `navigation`      | `FooterNavigation`                         | Enables the navigation floor with optional logo and required columns.                                  |
| `contacts`        | `FooterContacts`                           | Enables the contacts floor when it contains at least one link.                                         |
| `disclaimer`      | `FooterDisclaimer`                         | Enables the disclaimer floor. The text is rendered through YFM.                                        |
| `copyright`       | `FooterCopyright`                          | Enables the copyright floor with links, optional language switcher, optional logo, and copyright text. |
| `attribution`     | `boolean`                                  | Enables the attribution floor.                                                                         |
| `backgroundColor` | `ThemeSupporting<string>`                  | Theme-aware footer background color.                                                                   |
| `indentTop`       | `'0' \| 'xs' \| 's' \| 'm' \| 'l' \| 'xl'` | Top indent of the footer block.                                                                        |

## Shared types

```ts
type ThemeSupporting<T> =
    | T
    | {
          light: T;
          dark: T;
      };
```

```ts
type FooterLogo = {
    image: ThemeSupporting<ImageProps>;
    href?: string;
    alt?: string;
};
```

`FooterLogo.image` is resolved for the active theme. If `href` is provided, the logo is rendered as a link. `alt` is passed to the image and defaults to an empty string when omitted.

## Floor 1: Navigation

Use `navigation` when the footer needs product, documentation, company, or resource link columns. The navigation floor can also display a logo either to the left of the columns or above them.

### Type

```ts
type FooterNavigation = {
    logo?: FooterLogo & {
        placement?: 'left' | 'top';
    };
    colSizes?: ContainerSizes;
    columnsPerRow?: number;
    columns: FooterLinkColumn[];
};
```

```ts
type FooterLinkColumn = {
    title: string;
    links: LinkProps[];
};
```

### Props

| Prop             | Type                                         | Required | Description                                                                   |
| ---------------- | -------------------------------------------- | -------- | ----------------------------------------------------------------------------- |
| `columns`        | `FooterLinkColumn[]`                         | Yes      | Link columns rendered in the navigation floor.                                |
| `logo`           | `FooterLogo & {placement?: 'left' \| 'top'}` | No       | Optional footer logo.                                                         |
| `logo.placement` | `'left' \| 'top'`                            | No       | Logo layout. Defaults to `'left'`.                                            |
| `colSizes`       | `ContainerSizes`                             | No       | Grid sizes for every navigation column. Supports `all`, `sm`, `md`, and `lg`. |
| `columnsPerRow`  | `number`                                     | No       | Fixed number of columns per row. Schema allows values from `1` to `12`.       |

### Layout defaults

- Without custom `colSizes`, navigation columns use `{all: 6, sm: 4, md: 3, lg: 3}`.
- With `logo.placement: 'left'`, the logo occupies `{all: 12, md: 4}` and columns are placed next to it on wider screens.
- With `logo.placement: 'top'`, the logo is rendered above the columns and spans the full row.
- `columnsPerRow` switches the navigation row to a fixed percentage width per column: `100 / columnsPerRow`.

See stories:

- **ColumnsLayout** for `colSizes` and `columnsPerRow` examples.
- **LogoPlacement** for `logo.placement` examples.

### JSON example

```json
{
    "type": "footer-block",
    "navigation": {
        "logo": {
            "image": {
                "light": {"src": "story-assets/icon_1-with-text_light.svg"},
                "dark": {"src": "story-assets/icon_1-with-text_dark.svg"}
            },
            "href": "/",
            "alt": "Example company",
            "placement": "top"
        },
        "columnsPerRow": 4,
        "columns": [
            {
                "title": "Product",
                "links": [
                    {"text": "Overview", "url": "#overview"},
                    {"text": "Docs", "url": "#docs"}
                ]
            },
            {
                "title": "Company",
                "links": [
                    {"text": "About", "url": "#about"},
                    {"text": "Careers", "url": "#careers"}
                ]
            }
        ],
        "colSizes": {"all": 12, "sm": 6, "md": 3, "lg": 3}
    }
}
```

## Floor 2: Contacts

Use `contacts` for social media, community, or external profile links. Each link is rendered as an icon link that opens in a new tab with `rel="noopener noreferrer"`.

### Type

```ts
type FooterContacts = {
    title?: string;
    iconsSize?: 'm' | 'l';
    links?: FooterContactItem[];
    linksPosition?: 'left' | 'center';
    titlePosition?: 'top' | 'near';
};
```

```ts
type FooterContactItem = {
    icon: ThemeSupporting<ImageProps>;
    url: string;
    urlTitle?: string;
};
```

### Props

| Prop               | Type                  | Required | Description                                                                          |
| ------------------ | --------------------- | -------- | ------------------------------------------------------------------------------------ |
| `title`            | `string`              | No       | Contacts floor heading.                                                              |
| `links`            | `FooterContactItem[]` | No       | Social/contact links. The floor is not rendered when this array is missing or empty. |
| `iconsSize`        | `'m' \| 'l'`          | No       | Icon size. Defaults to `'l'`.                                                        |
| `linksPosition`    | `'left' \| 'center'`  | No       | Horizontal alignment of icon links. Defaults to `'left'`.                            |
| `titlePosition`    | `'top' \| 'near'`     | No       | Title placement relative to icons. Defaults to `'top'`.                              |
| `links[].urlTitle` | `string`              | No       | Used as the image `alt` text and as tooltip content for the icon link.               |

See stories:

- **ContactsPositions** for `linksPosition` and `titlePosition` combinations.
- **ContactsIconsSize** for `iconsSize` examples.
- **ContactsUrlTitles** for `urlTitle` tooltip behavior.

### JSON example

```json
{
    "type": "footer-block",
    "contacts": {
        "title": "Join us",
        "iconsSize": "l",
        "linksPosition": "center",
        "titlePosition": "near",
        "links": [
            {
                "icon": {
                    "light": {"src": "story-assets/icons-github_light.svg"},
                    "dark": {"src": "story-assets/icons-github_dark.svg"}
                },
                "url": "https://example.com",
                "urlTitle": "GitHub"
            }
        ]
    }
}
```

## Floor 3: Disclaimer

Use `disclaimer` for legal notices, trademark information, licensing details, or other long-form text. The text is rendered with `YFMWrapper`, so Markdown/YFM links and inline formatting are supported.

### Type

```ts
type FooterDisclaimer = {
    text: string;
    align: 'left' | 'center';
};
```

### Props

| Prop    | Type                 | Required | Description                                              |
| ------- | -------------------- | -------- | -------------------------------------------------------- |
| `text`  | `string`             | Yes      | Disclaimer text rendered as YFM.                         |
| `align` | `'left' \| 'center'` | Yes      | Disclaimer text alignment. Runtime fallback is `'left'`. |

See story: **DisclaimerAlignment**.

### JSON example

```json
{
    "type": "footer-block",
    "disclaimer": {
        "text": "Legal disclaimer text with a [link](https://example.com).",
        "align": "left"
    }
}
```

## Floor 4: Copyright

Use `copyright` for legal links, language selection, copyright text, and an optional logo. The floor adapts its layout depending on which parts are provided.

### Type

```ts
type FooterCopyright = {
    links?: Array<{text: string; url: string}>;
    linksOverflowStrategy?: 'dropdown' | 'line-wrap';
    languageSwitcher?: FooterLanguageSwitcher;
    logo?: FooterLogo;
    copyrightText?: string;
};
```

```ts
type FooterLanguageSwitcher = {
    items: FooterLanguageSwitcherItem[];
    buttonText?: string;
    image?: ThemeSupporting<ImageProps>;
};
```

```ts
type FooterLanguageSwitcherItem = {
    text: string;
    href: string;
};
```

### Props

| Prop                          | Type                                 | Required                             | Description                                            |
| ----------------------------- | ------------------------------------ | ------------------------------------ | ------------------------------------------------------ |
| `links`                       | `Array<{text: string; url: string}>` | No                                   | Legal or service links.                                |
| `linksOverflowStrategy`       | `'dropdown' \| 'line-wrap'`          | No                                   | Overflow behavior for links. Defaults to `'dropdown'`. |
| `languageSwitcher`            | `FooterLanguageSwitcher`             | No                                   | Dropdown language switcher rendered on the right side. |
| `languageSwitcher.items`      | `FooterLanguageSwitcherItem[]`       | Yes, when `languageSwitcher` is used | Language items converted to navigation links.          |
| `languageSwitcher.buttonText` | `string`                             | No                                   | Dropdown button text. Defaults to `'Language'`.        |
| `languageSwitcher.image`      | `ThemeSupporting<ImageProps>`        | No                                   | Optional theme-aware icon for the language switcher.   |
| `logo`                        | `FooterLogo`                         | No                                   | Optional logo for the copyright floor.                 |
| `copyrightText`               | `string`                             | No                                   | Copyright text rendered on the right side.             |

### Layout and overflow behavior

- `linksOverflowStrategy: 'dropdown'` keeps fitting links visible and moves overflowing links into a dropdown on desktop.
- `linksOverflowStrategy: 'line-wrap'` lets the links wrap to additional lines.
- When only links are present, links are centered.
- When `logo`, `links`, and right-side content (`languageSwitcher` or `copyrightText`) are all present, the floor uses a three-column layout: logo on the left, links in the center, right-side content on the right.
- On smaller widths, the layout stacks responsively.

See stories:

- **LinksOverflowStrategy** for dropdown and wrapping behavior.
- **LinksAlignment** for automatic layout alignment.
